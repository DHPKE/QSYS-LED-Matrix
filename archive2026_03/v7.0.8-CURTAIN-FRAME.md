# v7.0.8 — Curtain Frame Mode Update

**Date:** 2026-03-04  
**Commit:** `1ec9a71`  
**Branch:** `v7-curtain-mode`

## Change Summary

Changed curtain indicator mode from **3-pixel left/right bars** to **2-pixel frame around entire display**.

### Before (v7.0.0 - v7.0.7)
```
┌───┬────────────────────────────────────────────────────┬───┐
│ L │                                                    │ R │
│ E │          Display content here                     │ I │
│ F │          (58 pixels wide)                         │ G │
│ T │                                                    │ H │
│   │                                                    │ T │
│ 3 │                                                    │ 3 │
│ p │                                                    │ p │
│ x │                                                    │ x │
└───┴────────────────────────────────────────────────────┴───┘
```
- Left bar: pixels 0-2 (3 pixels wide, full height)
- Right bar: pixels 61-63 (3 pixels wide, full height)
- Middle area: pixels 3-60 (58 pixels usable)

### After (v7.0.8+)
```
┌──────────────────────────────────────────────────────────┐
│  2-pixel top border                                      │
├──┬────────────────────────────────────────────────────┬──┤
│2 │                                                    │2 │
│p │          Display content here                     │p │
│x │          (entire 64x32 area)                      │x │
│  │                                                    │  │
├──┴────────────────────────────────────────────────────┴──┤
│  2-pixel bottom border                                   │
└──────────────────────────────────────────────────────────┘
```
- Top edge: pixels (0,0) to (63,1) — 2 pixels high
- Bottom edge: pixels (0,30) to (63,31) — 2 pixels high
- Left edge: pixels (0,0) to (1,31) — 2 pixels wide
- Right edge: pixels (62,0) to (63,31) — 2 pixels wide
- Inner area: (2,2) to (61,29) — 60×28 pixels

## Technical Changes

### File: `rpi/curtain_manager.py`

**Old render logic:**
```python
# Left curtain bar (pixels 0-2, full height)
draw.rectangle(
    [(0, 0), (CURTAIN_WIDTH - 1, MATRIX_HEIGHT - 1)],
    fill=color
)

# Right curtain bar (pixels 61-63, full height)
right_x = MATRIX_WIDTH - CURTAIN_WIDTH
draw.rectangle(
    [(right_x, 0), (MATRIX_WIDTH - 1, MATRIX_HEIGHT - 1)],
    fill=color
)
```

**New render logic:**
```python
# Draw 2-pixel frame around entire display
# Top edge (2 pixels high)
draw.rectangle(
    [(0, 0), (MATRIX_WIDTH - 1, 1)],
    fill=color
)

# Bottom edge (2 pixels high)
draw.rectangle(
    [(0, MATRIX_HEIGHT - 2), (MATRIX_WIDTH - 1, MATRIX_HEIGHT - 1)],
    fill=color
)

# Left edge (2 pixels wide, full height)
draw.rectangle(
    [(0, 0), (1, MATRIX_HEIGHT - 1)],
    fill=color
)

# Right edge (2 pixels wide, full height)
draw.rectangle(
    [(MATRIX_WIDTH - 2, 0), (MATRIX_WIDTH - 1, MATRIX_HEIGHT - 1)],
    fill=color
)
```

### File: `rpi/config.py`

**Changed constants:**
```python
# OLD
CURTAIN_WIDTH = 3          # Width of each curtain bar in pixels
CURTAIN_AUTO_REMAP = True  # True = auto-shift segments

# NEW
CURTAIN_WIDTH = 2          # Width of frame border in pixels
CURTAIN_AUTO_REMAP = False # False = segments stay as-is, frame overlays
```

**Updated documentation:**
- Removed mention of "left/right bars" and "3 pixels"
- Added explanation of 2-pixel frame on all four edges
- Clarified that segments no longer auto-remap (frame is overlay)
- Updated inner display area coordinates: (2,2) to (61,29)

### File: `rpi/main.py`

**Version bump:**

## Behavior Changes

### Segment Rendering
- **Before:** Segments auto-shifted when curtain active (x+3, width-6)
- **After:** Segments render at original positions, frame overlays on top

### Visual Appearance
- **Before:** Two vertical bars flanking content (theater curtain style)
- **After:** Complete frame border (picture frame style)

### Inner Display Area
- **Before:** 58×32 pixels (64 - 3 - 3)
- **After:** 60×28 pixels (64 - 2 - 2, 32 - 2 - 2)
  - **Gain:** +2 pixels width
  - **Loss:** -2 pixels height

### Z-Index
- **Same:** Frame always renders on top of segments (highest z-index)

## Use Cases

### Group-Based Indicators
Curtain frame can be configured per group (1-8) to visually indicate:
- Active speaker zones
- Emergency alerts (red frame)
- Meeting room status (green/red frame)
- Broadcast vs local content
- Source/routing indicators

### Color-Coded Frames
Configure different colors per group:
```json
{"cmd": "curtain_config", "group": 1, "enabled": true, "color": "FF0000"}  // Red frame
{"cmd": "curtain_config", "group": 2, "enabled": true, "color": "00FF00"}  // Green frame
{"cmd": "curtain_config", "group": 3, "enabled": true, "color": "0000FF"}  // Blue frame
```

Then toggle visibility:
```json
{"cmd": "curtain_visibility", "group": 1, "visible": true}   // Show red frame
{"cmd": "curtain_visibility", "group": 1, "visible": false}  // Hide frame
```

## Deployment

### Update Files on Raspberry Pi

```bash
# SSH into the Pi
ssh pi@10.1.1.25

# Stop the service
sudo systemctl stop led-matrix

# Backup current version
sudo cp -r /opt/led-matrix /opt/led-matrix.backup.v7.0.7

# Copy updated files
sudo cp curtain_manager.py /opt/led-matrix/
sudo cp config.py /opt/led-matrix/
sudo cp main.py /opt/led-matrix/

# Restart service
sudo systemctl start led-matrix

# Check logs
sudo journalctl -u led-matrix -f
```

### Verify Version

```bash
# Check version in log output
sudo journalctl -u led-matrix --since "1 minute ago" | grep Version

# Expected output:
# Version: 7.0.8 (Curtain Frame Mode)
```

## Q-SYS Plugin Compatibility

The Q-SYS plugin (v7.1.0) does **not** need updating — it sends the same curtain commands:

```lua
-- Curtain config (unchanged)
{"cmd": "curtain_config", "group": N, "enabled": true/false, "color": "RRGGBB"}

-- Curtain visibility toggle (unchanged)
{"cmd": "curtain_visibility", "group": N, "visible": true/false}
```

The visual change is purely in the Python renderer — Q-SYS plugin UI and commands remain identical.
